# MorphOS Phase 0 Foundation Report

## Executive Summary

MorphOS is an AI-native operating system for dynamic workspace generation. It shifts the paradigm from static UI components to dynamically generated, data-driven interfaces. At its core, MorphOS leverages a strictly defined JSON Abstract Syntax Tree (AST) generated by an AI engine (OpenAI), persisted via a real-time backend (Convex), and rendered by a stateless frontend renderer. Phase 0 has successfully established the foundational architecture, design principles, and comprehensive documentation needed to begin implementation.

## Core Development Principles

Throughout development, every implementation decision must follow these principles.

### 1. JSON AST First
The JSON AST is the single contract between AI, Convex, and the Renderer.

### 2. AI Generates Data, Not UI
The AI never generates HTML, JSX, or React components.
It only generates valid JSON AST.

### 3. Renderer Is Stateless
The renderer receives validated JSON AST and renders it.
It never contains business logic.

### 4. Convex Is the Single Source of Truth
Application state is always stored in Convex.
No duplicated application state should exist in the frontend.

### 5. Widgets Are Pure Components
Widgets are presentation-only.
They receive props and emit events.
They never contain AI logic or database logic.

### 6. Validation Before Rendering
Every JSON AST must pass Zod validation before it reaches the renderer.

### 7. Modular Architecture
Every module should be replaceable without affecting the rest of the system.
